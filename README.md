# ⚽ Кубок друзей · ЧМ-2026

Турнир прогнозов на Чемпионат мира 2026 для компании друзей. Веб-сервис целиком
работает на **GitHub** (GitHub Pages + GitHub Actions), без сторонних баз данных —
поэтому доступен из РФ и бесплатен.

- Ставишь на **счёт матча** и **трёх авторов голов**.
- Результаты и забивших **бот тянет автоматически** с API-Football.
- Очки и таблица считаются сами, обновляются раз в час.
- Чужие ставки **скрыты до стартового свистка**; всё в репозитории зашифровано.
- Прогноз на **чемпиона** и **лучшего бомбардира** — с большими бонусами.
- Дизайн в духе ЧМ-2026, удобно с телефона.

> Полные правила начисления очков — на вкладке **«Правила»** внутри приложения.

---

## Как это устроено

```
Браузер (GitHub Pages, статика)
  • вход по логину+паролю → пароль расшифровывает «вшитый» GitHub-токен
  • ставка шифруется и коммитится файлом в data/bets/<user>/<matchId>.json

GitHub Action (раз в час, секреты на сервере)
  • тянет с API-Football счёт, авторов голов, составы
  • расшифровывает ставки приватным ключом бота, считает очки
  • после свистка раскрывает ставки матча для участников
  • пишет data/matches.json, standings.json, squads.json, revealed/*
```

Подробности по приватности и шифрованию — в разделе [Безопасность](#безопасность).

## Быстрый старт (один раз, ~15 минут)

### 1. Создай GitHub fine-grained токен
GitHub → **Settings → Developer settings → Fine-grained tokens → Generate new token**:
- **Resource owner** — твой аккаунт, **Repository access** — только репозиторий `lvs`.
- **Permissions → Repository → Contents: Read and write**.
- Срок — например, до 1 августа 2026.
- Скопируй токен (`github_pat_...`).

### 2. Получи бесплатный ключ API-Football
Зарегистрируйся на <https://dashboard.api-football.com> → скопируй **API key**
(бесплатный тариф: 100 запросов/день — нам хватает с запасом).

### 3. Впиши участников
Отредактируй [`config/participants.json`](config/participants.json) — массив имён (7–8 друзей).

### 4. Запусти настройку локально
```bash
npm install
GH_TOKEN="github_pat_ВАШ_ТОКЕН" node scripts/setup.mjs
```
Скрипт:
- сгенерирует ключи бота и общий ключ, создаст каждому **сильный пароль**;
- зашифрует токен под каждого участника → обновит `config/users.json` и `config/app.json`;
- создаст **`SECRETS.txt`** (он в `.gitignore`, в репозиторий не попадёт) с логинами/паролями
  и серверными секретами.

### 5. Добавь секреты в репозиторий
GitHub → **Settings → Secrets and variables → Actions → New repository secret**.
Возьми значения из `SECRETS.txt`:
- `ACTION_PRIVATE_KEY`
- `SHARED_KEY`
- `API_FOOTBALL_KEY` (ключ из шага 2)

### 6. Запушь и включи Pages
```bash
git add -A
git commit -m "Настройка турнира"
git push
```
GitHub → **Settings → Pages** → Source: **Deploy from a branch**, ветка `main`, папка `/ (root)`.
Сайт появится по адресу `https://<твой-логин>.github.io/lvs/`.

### 7. Первый запуск бота
GitHub → вкладка **Actions** → workflow **«Update results & standings»** → **Run workflow**.
Он подтянет реальное расписание ЧМ-2026 и заменит демо-данные. Дальше — раз в час сам.

### 8. Раздай друзьям
Каждому — ссылку на сайт и его **логин/пароль** из `SECRETS.txt` (лично, в личке).

## Локальный просмотр
```bash
npm run serve     # http://localhost:8080
```
Для локального входа нужно сначала выполнить `node scripts/setup.mjs` (создаст учётки).
Запись ставок локально не сработает без реального токена — это нормально, для записи
нужен прод (GitHub Pages) и валидный токен.

## Управление турниром

- **Рейтинг ФИФА** (для коэффициента сложности) — [`data/fifa-ranking.json`](data/fifa-ranking.json),
  ключ — английское название команды. Правь вручную при необходимости.
- **Ручные правки** — [`data/overrides.json`](data/overrides.json):
  переопределить множитель матча — `{ "matches": { "<fixtureId>": { "multiplier": 1.6 } } }`.
- **Очки и бонусы** настраиваются в `config/app.json` → `scoring` (значения за счёт, авторов,
  хет-трик, джекпот тура, бонусы за чемпиона/бомбардира) и `multipliers` (коэффициенты).
- **Лига/сезон** API — `config/app.json` → `api` (`leagueId: 1` — World Cup, `season: 2026`).
  Если бот не находит матчи, проверь эти значения в документации API-Football.

## Безопасность

- **Токен записи** хранится в репозитории только в **зашифрованном** виде (под каждого
  участника), поэтому GitHub его не обнаруживает и не отзывает. Расшифровывается в
  браузере паролём участника (Argon2id). В открытом виде токена в репо нет.
- **Ставки** шифруются так, что до начала матча их может прочитать только сам автор и
  бот (sealed box на ключ бота). После свистка бот раскрывает ставки матча, перешифровав
  их общим ключом друзей — их видят только участники, но не публика.
- Используй сильные пароли (их генерирует `setup.mjs`) и не коммить `SECRETS.txt`.

## Лимиты и заметки
- Бесплатный API-Football — 100 запросов/день. Бот бережёт квоту: расписание раз в час,
  события только для завершённых матчей, составы команд подгружаются партиями и кэшируются.
- Демо-данные в `data/*.json` — заглушка, чтобы сайт сразу работал; первый запуск Action
  заменяет их реальными.

## Структура
```
index.html                 # оболочка SPA
assets/css/styles.css       # дизайн ЧМ-2026
assets/js/                  # фронтенд (auth, crypto, github, bets, scoring, ui/*)
scripts/setup.mjs           # разовая настройка (локально)
scripts/update.mjs          # задача бота (GitHub Action)
.github/workflows/update.yml
config/app.json             # настройки, очки, множители, публичный ключ бота
config/users.json           # участники (шифртекст) — генерится setup.mjs
data/                       # matches, squads, standings, fifa-ranking, overrides, bets, revealed
```
